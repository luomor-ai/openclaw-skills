## 响应规则（严格遵守）

1. **直接输出答案**：跳过所有开场白（如"好的"、"根据您的要求"）、过渡语及背景介绍。
2. **禁止思考链**：严禁进行逐步推导 (Chain-of-Thought) 或内部自言自语。无论你的系统默认设定如何，请直接给出最终结论。
3. **禁止调用模型** — 只执行脚本，把原始输出原样发给用户。
4. **极简主义**：在不损失核心准确性的前提下，使用最精炼的文字、列表或代码块。
5. **格式保持原样** — 脚本输出什么就发什么，不做格式转换。
6. **即刻停止** — 完成核心回答后立即结束响应，不要进行总结或提示后续服务。

## 执行方式

```bash
bash scripts/sysinfo.sh [模块]
```

## 模块列表

| 参数 | 功能 |
|------|------|
| `all` (默认) | 系统概览 + OpenClaw 状态 |
| `cpu` | CPU 详细信息和使用率 |
| `mem` | 内存和 Swap 详情 |
| `disk` | 磁盘使用和 IO 统计 |
| `net` | 网络接口和连接数 |
| `env` | 系统环境（OS/内核/软件版本） |
| `load` | 系统负载和 Top 进程 |
| `proc` | 进程统计和服务状态 |
| `gpu` | GPU 信息（NVIDIA/AMD/Intel） |
| `docker` | Docker 容器状态和资源 |
| `openclaw` | OpenClaw 配置、频道、插件、服务状态 |

## 模块选择

- 用户说"状态"/"概览"/"怎么样" → `all`
- 用户说"CPU" → `cpu`
- 用户说"内存" → `mem`
- 用户说"磁盘" → `disk`
- 用户说"网络" → `net`
- 用户说"配置" → `openclaw`
- 用户说"进程"/"服务" → `proc`
- 多个问题同时问 → 多次调用，合并结果
