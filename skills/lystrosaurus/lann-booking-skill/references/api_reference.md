## 工具详细说明

### 1. create_booking 工具

**功能**: 创建蘭泰式按摩预约

**参数表**:
| 参数名 | 类型 | 必填 | 描述 | 格式要求 | 示例 |
|--------|------|------|------|----------|------|
| phone | string | 是 | 手机号码 | 11位中国大陆手机号 | "13800138000" |
| storeName | string | 是 | 门店名称 | 支持智能匹配 | "淮海店" |
| serviceName | string | 是 | 服务项目名称 | 支持智能匹配 | "传统古法全身按摩 -90分钟" |
| peopleCount | number | 是 | 预约人数 | 1-20人 | 2 |
| bookingTime | string | 是 | 预约时间 | ISO 8601格式 | "2024-01-15T14:00:00" |

**响应字段**:
```json
{
  "success": true,
  "bookingId": "BK1704012345678A1B2",
  "message": "预约成功！已为您发送确认短信。",
  "storeInfo": {
    "name": "门店名称",
    "address": "详细地址",
    "telephone": "联系电话",
    "traffic": "交通指引"
  },
  "serviceInfo": {
    "name": "服务名称",
    "desc": "服务描述"
  }
}
```

**错误响应**:
```json
{
  "success": false,
  "error": "错误描述",
  "details": "详细错误信息"
}
```

## 数据格式要求

### 手机号格式
- 必须为11位数字
- 中国大陆手机号格式
- 示例: "13800138000"

### 时间格式
- ISO 8601格式
- 示例: "2024-01-15T14:00:00"
- 必须为未来30天内的时间
- 建议使用24小时制

### 智能匹配规则
1. **精确匹配**: 完全相同的字符串
2. **包含匹配**: 输入是目标字符串的子串
3. **相似度匹配**: 编辑距离小于3的字符串

## 常见问题解答

### Q1: MCP服务器连接失败怎么办？
1. 检查是否已安装: `npm list lann-mcp-server`
2. 检查MCP配置: 确保在Claude Desktop配置中正确添加了lann服务器
3. 尝试重新启动: 重启Claude Desktop应用

### Q2: 预约时间格式错误怎么办？
- 确保使用ISO 8601格式: YYYY-MM-DDTHH:MM:SS
- 示例: "2024-01-15T14:00:00" 表示2024年1月15日下午2点
- 可以使用在线工具验证时间格式

### Q3: 门店或服务找不到怎么办？
1. 尝试更宽泛的关键词
2. 检查城市名称是否正确
3. 询问用户具体需求，提供建议
4. 使用智能匹配功能

### Q4: 预约人数限制是多少？
- 最小: 1人
- 最大: 20人
- 建议: 超过10人建议提前电话预约

### Q5: 可以预约多久之后的服务？
- 最早: 当前时间后1小时
- 最晚: 当前时间后30天
- 建议: 提前1-7天预约

## 故障排除

### 错误 1: "MCP server 'lann' not found"
**原因**: MCP 服务器未正确配置
**解决方案**:
- 检查 Skill Hub 或 MCP 配置文件，确保 lann-mcp-server 已正确添加
- 确认配置格式正确:
```json
{
  "mcpServers": {
    "lann": {
      "command": "npx",
      "args": ["lann-mcp-server"]
    }
  }
}
```

### 错误2: "Invalid phone number format"
**原因**: 手机号格式不正确
**解决方案**:
- 确保是11位数字
- 确保没有空格或特殊字符
- 示例: "13800138000"

### 错误 3: "Booking time must be within next 30 days"
**原因**: 预约时间超出限制
**解决方案**:
- 检查预约时间是否在未来 30 天内
- 使用当前时间计算验证

### 错误 4: "Store not found"
**原因**: 门店名称不匹配
**解决方案**:
1. 先调用 `query_stores` 工具查询可用门店
2. 从查询结果中选择正确的门店名称
3. 使用智能匹配功能
