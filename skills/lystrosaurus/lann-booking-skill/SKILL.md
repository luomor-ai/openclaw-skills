# Lann预约技能

## Skill Hub 兼容性

本技能完全符合 Skill Hub 标准技能规范:
- ✅ 通过标准 MCP 协议通信
- ✅ 无特定 CLI 工具依赖
- ✅ 业务逻辑与通信层解耦
- ✅ 可在 Skill Hub 环境中直接加载

## 概述

本技能提供使用Lann MCP服务器进行泰式按摩门店查询、服务查询和预约创建的完整指南。通过集成MCP服务器，您可以快速访问蘭泰式按摩的门店信息、服务目录，并创建预约订单。

## 核心功能

### 1. 门店查询
- 支持按城市（上海、杭州、成都等）搜索门店
- 支持按关键词（如"淮海"、"新天地"、"地铁"等）搜索门店
- 获取门店详细信息：地址、电话、交通指引

### 2. 服务查询  
- 支持按关键词（如"精油"、"古法"、"拉伸"、"面部"等）搜索服务
- 支持按服务时长（60、90、120分钟等）筛选服务
- 获取服务详细信息：名称、描述、时长

### 3. 预约创建
- 智能匹配门店和服务名称（支持模糊匹配）
- 创建预约订单并获取确认信息
- 支持多人预约（1-20人）
- 发送确认短信给用户

## MCP服务器安装与配置

### 安装MCP服务器
在使用本技能前，需要先安装Lann MCP服务器：

```bash
npm install lann-mcp-server
```

### 配置 MCP 服务器
在 Skill Hub 或支持 MCP 的环境中，通过以下配置连接 Lann MCP 服务器:

```json
{
  "mcpServers": {
    "lann": {
      "command": "npx",
      "args": ["lann-mcp-server"]
    }
  }
}
```

环境会自动发现并加载已配置的 MCP 服务器，无需手动检查或列出工具。

## 可用工具说明

### 1. `create_booking` - 创建预约
创建蘭泰式按摩预约。

**参数要求：**
- `phone` (必填): 11位中国大陆手机号
- `storeName` (必填): 门店名称（支持智能匹配）
- `serviceName` (必填): 服务项目名称（支持智能匹配）
- `peopleCount` (必填): 预约人数（1-20人）
- `bookingTime` (必填): 预约时间（ISO 8601格式，如"2024-01-15T14:00:00"）

**调用示例**:
通过标准 MCP 协议调用此工具，传递以下参数:
```json
{
  "phone": "13800138000",
  "storeName": "淮海店",
  "serviceName": "传统古法全身按摩 -90 分钟",
  "peopleCount": 2,

}
```

## 工作流程指南

### 场景1：用户需要查询门店信息
**典型请求：** "帮我查一下上海的门店"、"淮海店在哪里？"

1. **确定查询条件**：
   - 城市（如"上海"）
   - 关键词（如"淮海"、"新天地"）

2. **调用查询工具**:
   调用 `query_stores` 工具，传入参数:
   - city: "上海"
   - keyword: "淮海"
   
   Skill Hub 将自动调用 MCP 服务器并返回结果。

3. **格式化响应**：
   - 提取门店信息
   - 显示地址、电话、交通指引
   - 提供后续操作建议

### 场景2：用户需要查询服务项目
**典型请求：** "有什么90分钟的服务？"、"我想做精油按摩"

1. **确定查询条件**：
   - 服务类型关键词（如"精油"、"古法"）
   - 服务时长（如60、90、120分钟）

2. **调用查询工具**:
   调用 `query_services` 工具，传入参数:
   - keyword: "精油"
   - duration: 90
   
   Skill Hub 将自动调用 MCP 服务器并返回结果。

3. **格式化响应**：
   - 列出符合条件的服务项目
   - 显示服务描述和时长
   - 提供预约建议

### 场景3：用户需要创建预约
**典型请求：** "我想预约淮海店的90分钟古法按摩"、"帮我预约2个人明天下午2点的按摩"

1. **收集预约信息**：
   - 手机号码（11位）
   - 门店名称（支持模糊匹配）
   - 服务名称（支持模糊匹配）
   - 预约人数（1-20人）
   - 预约时间（ISO 8601格式）

2. **验证信息完整性**：
   - 确保所有必填字段都有值
   - 验证手机号格式
   - 验证预约时间在未来30天内

3. **创建预约**:
   调用 `create_booking` 工具，传入完整参数:
   ```json
   {
     "phone": "13800138000",
     "storeName": "淮海店",
     "serviceName": "传统古法全身按摩 -90 分钟",
     "peopleCount": 2,
     "bookingTime": "2024-01-15T14:00:00"
   }
   ```
   
   Skill Hub 将自动调用 MCP 服务器并返回结果。

4. **处理响应**：
   - 显示预约确认信息
   - 提供预约ID
   - 显示门店和服务详细信息
   - 确认短信发送状态

## 智能匹配说明

Lann MCP服务器实现了智能匹配算法，支持以下匹配方式：

### 1. 精确匹配
- 完全相等的匹配（高置信度）
- 示例：输入"淮海店" → 匹配到"淮海店"

### 2. 包含匹配
- 输入是门店/服务名称的子串（高置信度）
- 示例：输入"淮海" → 匹配到"淮海店"

### 3. 相似度匹配
- 基于编辑距离的模糊匹配（中/低置信度）
- 示例：输入"古法按摩" → 可能匹配到"传统古法全身按摩 -90分钟"

**提示用户**：提供模糊匹配建议时，可以询问用户是否确认匹配结果。

## 错误处理

### 常见错误及解决方案

1. **MCP 服务器未连接**
   - 症状：无法调用 Lann 相关工具
   - 解决方案：检查 MCP 服务器配置，确保 lann-mcp-server 已正确安装和配置

2. **参数验证失败**
   - 症状：返回参数验证错误信息
   - 解决方案：检查必填字段、格式要求（特别是手机号和日期格式）

3. **门店/服务未找到**
   - 症状：查询结果为空
   - 解决方案：尝试更宽泛的关键词，或询问用户具体需求

4. **预约时间限制**
   - 症状：只能预约未来30天内的服务
   - 解决方案：建议用户调整预约时间

## 最佳实践

### 1. 查询优先原则
在创建预约前，建议先查询门店和服务：
1. 查询用户所在城市的门店
2. 查询用户感兴趣的服务类型
3. 确认匹配结果后再创建预约

### 2. 信息验证
创建预约前验证：
- 手机号格式正确（11位数字）
- 预约时间格式正确（ISO 8601）
- 预约时间在未来30天内
- 预约人数在1-20人范围内

### 3. 用户友好提示
- 提供清晰的查询结果
- 解释智能匹配的结果
- 提供预约后的确认信息
- 建议保存预约ID以便查询
